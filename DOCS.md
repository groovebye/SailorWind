## 1. Обзор продукта

SailorWind — это операционный инструмент для шкипера, позволяющий:

- **Планировать переходы** с учётом погоды, приливов и ограничений лодки
- **Принимать решение GO / CAUTION / NO-GO** на основе прогноза ветра, волн и зыби
- **Видеть часовой расклад перехода** с режимом хода (парус/мотор/моторсейл), расходом топлива, комфортом
- **Использовать полярную модель лодки** для реалистичной оценки скорости под парусами
- **Редактировать маршрут** на карте с глубинами, контурами и морскими знаками
- **Сравнивать марины** с ценами, услугами и рекомендациями
- **Вести логбук перехода** с чекпоинтами, наблюдениями и сравнением план/факт
- **Экспортировать GPX** для Navionics / OpenCPN

## 3. Страницы приложения

### 3.1 Главная страница (`/`)

- Заголовок «SailorWind» с кнопкой создания нового пассажа
- Список последних 10 пассажей (название, дата отхода, кол-во точек)
- Каталог портов и марин: грид карточек по портовым зонам
  - Регион, количество марин, причальных мест
  - Минимальная цена стоянки (LOA 9.5m, низкий сезон)
  - Иконки: топливо, ремонт, рекомендации, риск орок

### 3.2 Мастер создания пассажа (`/new`)

**Шаг 1 — Маршрут:**
- Выбор порта отхода / прихода (выпадающие списки, мысы исключены)
- Дата и время отхода (по умолчанию: завтра 08:00)
- Скорость (1–15 узлов, шаг 0.5)
- Режим: «Дневные переходы» или «Нон-стоп»
- Модель погоды: ECMWF IFS 0.25°, ICON-EU, GFS, AROME France

**Шаг 2 — Промежуточные точки:**
- Все порты между стартом и финишем, отсортированные по coastlineNm
- Автоматически отмечены: старт, финиш, мысы, марины
- Ручное включение/выключение промежуточных остановок
- Предупреждения: >50 NM или >10 часов — жёлтый/красный индикатор
- Расчёт дистанции и времени для каждого участка

### 3.3 Панель пассажа (`/p/[id]`)

Главная страница запланированного перехода.

**Редактируемые параметры** (автосохранение с debounce 500ms):
- Дата/время отхода
- Скорость (узлы)
- Режим (дневной / нон-стоп)
- Модель погоды

**Переключатель источника погоды:** Open-Meteo ↔ Windy. Данные обоих источников кешируются, переключение без перезагрузки.

**Таблица прогноза по маршруту:**
- Группировка по дням/переходам
- Для каждой точки: ETA, эмодзи погоды, ветер (направление + узлы + Бофорт), порывы, волны (высота/период/направление), зыбь, мощность волн (kW/m), вердикт

**Раскрываемый подробный прогноз:** почасовые данные по каждой точке с подсветкой строк около ETA.

**Тайлы переходов:** кликабельные карточки каждого дневного перехода с вердиктом, открывают детальную страницу перехода.

### 3.4 Страница перехода (`/p/[id]/leg/[legIndex]`)

Основной операционный интерфейс шкипера. ~1900 строк кода, 22 секции.

**Закреплённая шапка (sticky bar):**
- Название перехода: «Day 1: Хихон → Луарка (47 NM, ~9.4h)»

## 4. Погодная система

### 4.1 Источники данных

| Источник | Данные | Модель | Стоимость |
|----------|--------|--------|-----------|
| Open-Meteo Weather | Ветер, порывы, осадки, облачность | ECMWF, GFS, ICON, AROME | Бесплатно |
| Open-Meteo Marine | Волны, зыбь (высота/период/направление) | ERA5 marine | Бесплатно, ~9 дней |
| Windy GFS | Ветер, порывы, осадки, температура | GFS | Бесплатно (API key) |
| Windy gfsWave | Волны, зыбь 1+2 | GFS Wave | Бесплатно, полные 10 дней |

### 4.2 Пороги GO / CAUTION / NO-GO

| Параметр | Обычная точка | Мыс |
|----------|--------------|-----|
| Ветер CAUTION | >20 узлов | >15 узлов |
| Ветер NO-GO | >30 узлов | >25 узлов |
| Порывы NO-GO | >35 узлов | >30 узлов |
| Волны CAUTION | >2.5 м | >2.0 м |
| Волны NO-GO | >3.5 м | >3.0 м |

### 4.3 Мощность волн (Wave Power)

Формула: `P = 0.5 × H² × T` (kW/m), где H — высота волны, T — период.

| kW/m | Категория | Цвет |
|------|-----------|------|
| <5 | Calm | Зелёный |
| 5–15 | Moderate | Синий |
| 15–30 | Rough | Жёлтый |
| ≥30 | Severe | Красный |

### 4.4 Встроенные карты прогноза

4 Windy iframe-эмбеда прямо на странице перехода:
- **Ветер** — анимированные стрелки направления и скорости
- **Волны** — высота волн с цветовым кодированием
- **Зыбь** — направление и период зыби
- **Осадки** — прогноз дождя

Каждая карта центрирована на середине перехода, zoom 8, с кнопкой «open ↗» для полного Windy.

### 4.5 Сдвиг координат для риасов

Порты внутри риасов (Ферроль, Виверо) возвращают null от морских API (попадают на сушу в сетке ~5км). Решение: координаты для морского API сдвигаются на +0.05° по широте (~3 NM на север) в открытое море.

## 5. Полярная модель лодки

### 5.1 Что это

Полярная диаграмма описывает скорость лодки в зависимости от силы ветра (TWS) и угла к ветру (TWA). Это позволяет реалистично оценивать скорость под парусами вместо грубых эвристик.

### 5.2 Структура данных

```
PolarData:
  twsKnots:   [6, 8, 10, 12, 16, 20, 25]     — 7 значений силы ветра
  twaDegrees: [40, 50, 60, 75, 90, 110, 135, 150, 165] — 9 углов к ветру
  boatSpeeds: матрица 9×7                       — скорость лодки (узлы)
  hullSpeedKt: 6.3                              — максимальная скорость корпуса
  targetUpwindTwaDeg: 42°                       — оптимальный угол лавировки
  targetDownwindTwaDeg: 155°                    — оптимальный угол полного курса
```

### 5.3 Матрица скоростей Bossanova

|  | 6kt | 8kt | 10kt | 12kt | 16kt | 20kt | 25kt |
|--|-----|-----|------|------|------|------|------|
| **40° (крутой бейдевинд)** | 2.1 | 2.8 | 3.4 | 3.9 | 4.4 | 4.6 | 4.2 |
| **50° (бейдевинд)** | 2.6 | 3.4 | 4.2 | 4.7 | 5.2 | 5.3 | 4.8 |
| **60° (галфвинд)** | 3.0 | 3.9 | 4.7 | 5.3 | 5.7 | 5.8 | 5.3 |
| **75° (полный галфвинд)** | 3.3 | 4.2 | 5.1 | 5.6 | 6.0 | 6.1 | 5.6 |
| **90° (бакштаг, лучший курс)** | 3.5 | 4.5 | 5.4 | 5.9 | 6.3 | 6.3 | 5.8 |
| **110° (полный бакштаг)** | 3.3 | 4.3 | 5.2 | 5.7 | 6.1 | 6.2 | 5.7 |
| **135° (фордевинд)** | 3.0 | 3.9 | 4.8 | 5.3 | 5.8 | 5.9 | 5.4 |
| **150° (глубокий фордевинд)** | 2.6 | 3.5 | 4.3 | 4.9 | 5.4 | 5.5 | 5.0 |
| **165° (почти в корму)** | 2.2 | 3.0 | 3.8 | 4.4 | 4.9 | 5.0 | 4.5 |

> Это **assumed polars** — консервативная крейсерская модель. Не официальные данные производителя. Будут уточняться по реальным логам.

### 5.4 Интерполяция

**Билинейная интерполяция** по двум осям:
1. Найти 2 ближайших значения TWS (нижний/верхний) и долю между ними
2. Найти 2 ближайших значения TWA (нижний/верхний) и долю
3. Получить 4 угловых значения скорости из матрицы
4. Интерполировать по TWA, затем по TWS

Результат ограничен hull speed (6.3 узлов).

### 5.5 Факторы деградации

Полярная скорость корректируется на реальные условия:

| Фактор | Условие | Коэффициент |
|--------|---------|-------------|
| Волны | >1.5m | ×0.96 |
| Волны | >2.0m | ×0.92 |
| Порывы | Разброс >10kt | ×0.95 |
| 1-й риф | reef1 | ×0.90 |
| 2-й риф | reef2 | ×0.78 |

### 5.6 Эффективность

`efficiency% = (polarSpeed / bestSpeedAtThisTWS) × 100`

Используется для выбора режима хода:
- **<40%** → мотор (бессмысленно идти под парусом)
- **40–55%** → моторсейл (парус + двигатель)
- **≥55%** → парус

### 5.7 Фолбэк

Если полярных данных нет (старые записи в БД), система автоматически использует эвристический алгоритм `estimateSailSpeedHeuristic()` с фиксированными коэффициентами по курсам.

## 6. Расчёт перехода (Timeline Engine)

### 6.1 Модуль `passage-computation.ts`

900+ строк. Генерирует почасовой расклад перехода.

**Входные данные:**
- Маршрут (автоматический из графа или ручной)
- Прогноз погоды по каждой точке
- Приливные данные
- Профиль лодки с полярной моделью

**Выходные данные** (на каждый час):
- Позиция (lat/lon), расстояние от старта / до финиша
- Курс, ветер (направление, скорость, порывы), волны, зыбь
- TWA (угол к ветру), курс относительно ветра
- Полярная скорость (BSP), SOG (с учётом течения)
- Полярная эффективность %, целевой угол
- Режим: парус / мотор / моторсейл
- Конфигурация парусов, уровень рифления
- Расход топлива за час, кумулятивный расход
- Балл комфорта (0–100), категория, причины
- Предупреждения, заметки

### 6.2 Определение режима хода

Приоритет решений:
1. **Порт** (< 1.2 NM от начала/конца) → мотор
2. **Штиль** (ветер < 7 узлов) → мотор
3. **Полярная эффективность < 40%** → мотор
4. **Полярная эффективность 40–55%** → моторсейл
5. **Слишком круто к ветру** (< 38°) → моторсейл или мотор
6. **SOG < 4.5 узлов под парусом** → переключение на мотор (правило Bossanova)
7. Иначе → парус

### 6.3 Топливная модель

| Режим | Расход | Обороты | Скорость |
|-------|--------|---------|----------|
| Мотор (cruise) | 2.5 л/ч | 2500 RPM | 6.2 узлов |
| Моторсейл | 1.75 л/ч | 2000 RPM | — |
| Парус | 0 л/ч | — | по полярам |

- Бак: 60 литров
- Запас: отслеживается с индикатором OK / LOW / CRITICAL

### 6.4 Комфорт

Отдельная оценка от безопасности (GO ≠ Comfortable).

Шкала: **0–100 баллов** → Comfortable / Moderate / Bumpy / Demanding / Uncomfortable.

Факторы (с вычетом баллов):
- Волны >2.5m: −22, >1.8m: −12, >1.2m: −6
- Зыбь >2.0m: −10, >1.3m: −4
- Порывы +14kt: −12, +8kt: −6
- Бейдевинд: −10, слишком круто: −18
- Мотор в волну: −5
- Обход мыса: −10
- Заход в порт: −4
- Ночь: −10
- Сильное течение >1.2kt: −4

### 6.5 Конфигурация парусов

Автоматически определяется по режиму, курсу, ветру и рифлению:
- `Full main + genoa` — основной вариант
- `1 reef + genoa` / `2 reefs + reduced headsail` — при рифлении
- `Main steadying sail + engine` — моторсейл
- `Engine only` — чистый мотор

### 6.6 Кеширование

Результаты хранятся в таблице `LegComputation` и инвалидируются при изменении:
- Прогноза погоды (forecastSignature)
- Маршрута (routeSignature, включая ручные точки)
- Профиля лодки (vesselSignature, включая полярные данные)

## 7. Маршрутизация

### 7.1 Граф прибрежных точек

Ручной граф из 62 навигационных точек и 64 двунаправленных рёбер вдоль побережья Северной Испании.

Типы узлов:
- **Порты**: Хихон, Кандас, Авилес, Луарка, Рибадео, Виверо, Ферроль, Ла-Корунья и др.
- **Мысы**: Кабо Пеньяс, Эстака де Барес, Кабо Ортегаль (+ точки обхода)
- **Промежуточные**: точки выхода из порта, оффшорные повороты, входы в риас

### 7.2 Алгоритм

**Дейкстра** (кратчайший путь) с евклидовой дистанцией. Для особо сложных участков (Кабо Пеньяс, глубокие риас) используются вручную заданные маршруты с фиксированными точками обхода.

### 7.3 Ручное редактирование маршрута

На карте перехода:
- **Modify Route** → режим редактирования: клик на карту добавляет точку, клик на точку удаляет
- **Save** → сохранение в БД (`PassageLegRoute`), инвалидация кеша таймлайна
- **Reset to Auto** → удаление ручного маршрута, возврат к графу

### 7.4 Контуры глубин

Из EMODnet батиметрии, сгенерированные через GDAL:

| Глубина | Цвет | Толщина |
|---------|------|---------|
| 5m | Красный | 1.5px сплошная |
| 10m | Оранжевый | 1.2px сплошная |
| 20m | Жёлтый | 1px пунктир |
| 50m | Серый | 0.8px пунктир |
| 100m | Тёмно-серый | 0.6px пунктир |
| 200m | Ещё темнее | 0.5px пунктир |

## 8. Приливы и течения

### 8.1 Модель

Полусуточная гармоническая модель. Период: ~12ч 25мин. Опорная точка: HW Dover.

Для каждого порта задано:
- Смещение от HW Dover (часы)
- Диапазон сизигийного прилива (springs)
- Диапазон квадратурного прилива (neaps)

Интерполяция springs/neaps по фазе Луны (cos от дней после полнолуния / 29.53).

### 8.2 Приливные данные (18 портов)

### 8.3 Приливные течения

5 критических зон:

| Зона | Макс. скорость | Примечание |
|------|---------------|------------|

### 8.4 Точность

~15–30 минут — достаточно для планирования перехода. Не заменяет официальные таблицы приливов.

## 9. Порты и марины

### 9.1 Двухслойная модель данных

**Маршрутный слой** (`Port`) — навигационные точки:
- 19 портов + 3 мыса
- Координаты, тип, coastlineNm
- Удобства: топливо, вода, электричество, ремонт
- Используются движком расчёта

**Контентный слой** (`PortArea` → `MarinaOption` → `NearbyPlace`) — информация для шкипера:
- `PortArea` = город/гавань (8 зон)
- `MarinaOption` = конкретная марина (10 марин) с ценами, подходом, удобствами
- `NearbyPlace` = рестораны, магазины, аптеки (23 объекта)

### 9.2 Рекомендации марин

6 сценариев использования с ранжированием:
1. **Транзит** — быстрая остановка на ночь
2. **Бюджет** — минимальная цена
3. **Месяц** — долгая стоянка
4. **Ремонт** — доступ к мастерским и кран
5. **Провизия** — рядом магазины
6. **Комфорт** — лучшие удобства и рестораны

### 9.3 Зоны орок

Данные из Orca Iberica / GTOA:
- **Побережье Галисии** (42.5–43.5°N, 7.5–9.5°W): средний риск
- **Кабо Финистерре** (42.5–43.0°N, 8.8–9.5°W): высокий риск
- Отображаются полупрозрачными прямоугольниками на картах
- Рекомендация: приложение GT Orcas для отслеживания

## 10. Система исполнения и логбук

### 10.1 Жизненный цикл перехода

`planned` → `active` → `completed` / `aborted`

### 10.2 Чекпоинты

Быстрые отметки ключевых событий:
- **Departure** — отход
- **Cape** — прохождение мыса
- **Reef In / Reef Out** — взятие/снятие рифов
- **Bailout** — уход на запасной порт
- **Arrival** — прибытие
- **Custom** — произвольная отметка

Каждый чекпоинт: тип + заголовок + заметка + время.

### 10.3 Наблюдения

Запись реальных условий:
- Ветер: скорость, направление
- Волны, зыбь (высота)
- Видимость (NM), барометр (hPa)
- Состояние моря (текст)
- Конфигурация парусов
- Комфорт: Comfortable / Moderate / Bumpy / Demanding / Hard work
- Свободная заметка

### 10.4 Дебриф (план vs факт)

После завершения перехода:
- Сравнение времён отхода / прибытия (план vs факт)
- Дельта продолжительности
- Сравнение прогноза ветра/волн с наблюдённым
- Комфорт: запланированный vs реальный
- Автогенерация инсайтов для `LegMemory`

### 10.5 Обучающий слой

- **LegMemory** — заметки по переходу (категории: weather, timing, comfort, hazard, approach)
- **PortMemory** — заметки по порту (категории: approach, berth, shore, hazard)
- Формируются автоматически из дебрифа и вручную

## 11. Дополнительные системы

### 11.1 GPX экспорт

- Экспорт одного перехода или всего пассажа
- Формат: GPX 1.1 с waypoints и route
- Совместимость: Navionics, OpenCPN, Garmin

### 11.2 Оффлайн-кеш

- Кеширование данных недавно просмотренных переходов в `localStorage`
- 6 часов — свежие данные (зелёный индикатор)
- 24 часа — устаревшие данные (жёлтый индикатор)
- Автоочистка после 24 часов

### 11.3 Обслуживание лодки

Модель `MaintenanceItem`:
- Категории: engine, rigging, safety, hull, electronics, general
- Статусы: ok, due, overdue, done
- Дата очередного ТО, моточасы
- Приоритет: low, medium, high, critical

API `/api/readiness` — проверка готовности к выходу:
- Список просроченных и предстоящих работ
- 10 стандартных напоминаний (VHF тест, масло, трюмная помпа, жилеты и т.д.)
- Общий статус: ready / check_needed / not_ready

### 11.4 Фолбэк морских знаков

24 вручную отобранных знака (маяки, буи, кардинальные знаки) на случай недоступности OpenSeaMap. Индикатор: «⚠ OpenSeaMap down — showing N curated marks».

### 11.5 Вебкамеры

Windy Webcams API v3: камеры в радиусе 20 км от портов отхода и прихода. Превью + ссылка на Windy player.

### 11.6 Тёмная/светлая тема

CSS-переменные + ThemeProvider:
- Автосохранение в localStorage
- Переключение тайлов карты (CartoDB Dark ↔ Voyager)

## 15. Известные ограничения

| Ограничение | Пояснение |
|-------------|-----------|
| Полярные данные assumed | Не официальные полары производителя — будут уточняться по логам |
| Приливы ~15–30 мин | Гармоническая модель, не официальные таблицы |
| Данные орок advisory | Нет интеграции с live API |
| Контент только Хихон → Ла-Корунья | Марины, сервисы, лоция — только для первого отрезка |
| Windy embed может быть недоступен | embed.windy.com иногда возвращает 404 |
| Нет VPP-физики | Нет крена, дрейфа, лавировочного оптимизатора |
| Нет real-time инструментов | Нет интеграции с бортовыми приборами |
| 1 лодка | Профиль только Bossanova, мультилодочность не реализована |
